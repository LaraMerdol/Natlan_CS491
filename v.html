2. Project Description

This project is a web-based database management application that will be used by both the employees and customers of the shipping system. The aim of the project is to increase the efficiency of communication and planning between branches, employees, and customers.


There are three user types in our project: customer, courier, employee. Each user will create an account with a username and a password then each of them will log in to the system. The system enables customers to ship their package by either submitting it to the branch directly or calling a courier so that s/he can transfer the package to the branch. A package can be fragile, spoilable, or normal. The packaging material such as the bubble pouch will be determined by the package type. Customers will be limited to calling only 2 couriers to their location in a day so that their right to call a courier is not abused. There are 3 types of couriers; fast, heavy, and default. Fast couriers are a little bit more expensive and they carry small, light packages. Heavy couriers carry big and heavy packages while default couriers carry normal packages at an affordable price. Since customers can send a package, they can also receive packages. If customers want to receive their package at a certain time period in the day, they have the ability to state that. The system enables the customer to track their packages as well. There are branches in each location where 10 to 50 employees work. These employees are charged with assigning packages to couriers which a branch has delivered from other branches and an employee can assign at most 20 packages to each courier. There are two ways that a customer can receive his/her package; couriers transport packages to each customer’s location or the customer can go to the branch’s already determined pickup location to take the package in person. These locations can be any type of shop and the customer can choose which pick-up location they want to go to according to their closeness to that location. There is also another functionality that our system provides which is to make contracts with companies. Some companies can make deals with specific shipping systems and they can send their packages via the shipping system. Since many packages are transported on a daily basis, some may be malformed. So, in order not to lose the trust of one’s customers, the system enables them to decline the shipping and write a report about that incident. Also, some packages may get lost due to extraordinary factors and for the same reason, customers have the right to file a report about it. There is also an evaluation step for customers so that they can rank their shipping experience. Each courier is evaluated with points.


Because there are different user types and different request combinations, a well-structured database will be needed for this project. The database will allow the data to be organized more efficiently than a normal file system.


3. Requirements

  3.1. Functional

    Customer

  • Customers can send packages by calling a courier to their location.
  • Customers can send packages by handing them over to the branch themselves.
  • Customers can fill a report and decline the shipping for malfunctioned packages and lost packages as well.
  • Customers can state what time they want their cargo to be delivered.
  • Customers can track their cargo status.
  • Customers can rate and review the couriers so that their corresponding branches can see the couriers’ performance.
  • Customers can determine the package type.
  • Customers can choose courier type according to their package.
  • Customers can determine whether they want their package to be delivered by a courier or take it in person from a pickup location.

    Courier

  • Couriers deliver the package to their assigned branch.

    Employee

  • Employees that are working in a branch, transfer packages to destination branches.
  • Employees assign the packages to couriers to deliver to customers.

  3.2. Non-Functional

    Security

  • Address of the customers must be kept private from other customers.
  • Usernames and passwords will be private.
  • Customers will rank couriers anonymously.

    Usability

  • A user-friendly interface will be implemented so that everyone can use it without having any problem with the functionalities of the system. Each operation can be executed easily.

    Scalability

  • The database can be enlarged without losing performance.
  • The system is able to handle an increase in workload by not causing any performance degradation.

4. Limitations

  • Username cannot be changed after creating an account.
  • User passwords must include upper, lower characters with a numeric character(s).
  • Complaints about a malformed or lost package cannot exceed 500 characters.
  • Complaint reports about malformed and lost packages cannot be filed 15 days after the delivery date.
  • An employee can assign at most 20 packages to a courier per day.
  • There must be at least 10 and at most 50 employees in each branch.
  • Customers can call at most 2 couriers to send packages in a day.
  • Each package is priced out by its dimension and weight.
  • Fast couriers can carry up to 10 kg, heavy couriers can carry up to 1000 kg and default couriers can carry up to 250 kg.
